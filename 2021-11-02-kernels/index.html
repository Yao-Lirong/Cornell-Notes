Kernels

Cornell University
November 2, 2021

Linear classifiers are great, but what if there exists no linear decision boundary? We should observe that nonlinear classifier in our (usually low) dimensional space is just a linear classifier on a higher dimensional space. When we project this higher-dimensional linear classifier into our lower-dimensional space, it appears nonlinear. As it turns out, there is an elegant way to incorporate non-linearities into most linear classifiers.

Handcrafted Feature Expansion¶

We can make linear classifiers non-linear by expanding the existing features. Formally, for a data vector x∈Rd\mathbf{x}\in\mathbb{R}^dx∈Rd, we apply the transformation x→ϕ(x)\mathbf{x} \rightarrow \phi(\mathbf{x})x→ϕ(x) where ϕ(x)∈RD\phi(\mathbf{x})\in\mathbb{R}^Dϕ(x)∈RD. Usually D≫dD \gg dD≫d because we add dimensions that capture non-linear interactions among the original features. l

  • Advantage: It is simple, and your problem stays convex and well behaved. (i.e. you can still use your original gradient descent code, just with the higher dimensional representation)

  • Disadvantage: ϕ(x)\phi(\mathbf{x})ϕ(x) might be very high dimensional.

Consider the following example: x=(x1x2⋮xd)\mathbf{x}=\begin{pmatrix}x_1\\ x_2\\ \vdots \\ x_d \end{pmatrix}x=⎝⎛​x1​x2​⋮xd​​⎠⎞​, and define ϕ(x)=(1x1⋮xdx1x2⋮xd−1xd⋮x1x2⋯xd)\phi(\mathbf{x})=\begin{pmatrix}1\\ x_1\\ \vdots \\x_d \\ x_1x_2 \\ \vdots \\ x_{d-1}x_d\\ \vdots \\x_1x_2\cdots x_d \end{pmatrix}ϕ(x)=⎝⎛​1x1​⋮xd​x1​x2​⋮xd−1​xd​⋮x1​x2​⋯xd​​⎠⎞​.

This new representation, ϕ(x)\phi(\mathbf{x})ϕ(x), is very expressive and allows for complicated non-linear decision boundaries - but the dimensionality is extremely high D=2dD = 2^dD=2d. This makes our algorithm unbearable (and quickly prohibitively) slow.

The Kernel Trick¶

Gradient Descent with Squared Loss¶

The kernel trick is a way to get around this dilemma by learning a function in the much higher dimensional space, without explicitly computing the value of a single vector ϕ(x)\phi(\mathbf{x})ϕ(x) or ever computing the full vector w\mathbf{w}w. We will represent these values only with x\mathbf xx and yyy.

It is based on the following observation: If we use gradient descent with any one of our standard loss functions, the gradient is a linear combination of the input samples. For example, in squared loss:

ℓ(w)=∑i=1n(w⊤xi−yi)2\ell(\mathbf{w}) = \sum_{i=1}^n (\mathbf{w}^\top \mathbf{x}_i-y_i)^2ℓ(w)=i=1∑n​(w⊤xi​−yi​)2

Inner-Product Computation¶

Let’s go back to the previous example, ϕ(x)=(1x1⋮xdx1x2⋮xd−1xd⋮x1x2⋯xd)\phi(\mathbf{x})=\begin{pmatrix}1\\ x_1\\ \vdots \\x_d \\ x_1x_2 \\ \vdots \\ x_{d-1}x_d\\ \vdots \\x_1x_2\cdots x_d \end{pmatrix}ϕ(x)=⎝⎛​1x1​⋮xd​x1​x2​⋮xd−1​xd​⋮x1​x2​⋯xd​​⎠⎞​.

The inner product ϕ(x)⊤ϕ(z)\phi(\mathbf{x})^\top \phi(\mathbf{z})ϕ(x)⊤ϕ(z) can be formulated as:

ϕ(x)⊤ϕ(z)=1⋅1+x1z1+x2z2+⋯+x1x2z1z2+⋯+x1⋯xdz1⋯zd=∏k=1d(1+xkzk).\phi(\mathbf{x})^\top \phi(\mathbf{z})=1\cdot 1+x_1z_1+x_2z_2+\cdots +x_1x_2z_1z_2+ \cdots +x_1\cdots x_dz_1\cdots z_d=\prod_{k=1}^d(1+x_kz_k)\text{.}ϕ(x)⊤ϕ(z)=1⋅1+x1​z1​+x2​z2​+⋯+x1​x2​z1​z2​+⋯+x1​⋯xd​z1​⋯zd​=k=1∏d​(1+xk​zk​).
(9)

The sum of 2d2^d2d terms becomes the product of ddd terms. Define the kernel function k\mathsf kk as:

k(xi,xj)=ϕ(xi)⊤ϕ(xj)\mathsf{k}(\mathbf{x}_i,\mathbf{x}_j) =\phi(\mathbf{x}_i)^\top \phi(\mathbf{x}_j)k(xi​,xj​)=ϕ(xi​)⊤ϕ(xj​)
(10)

With a finite training set of nnn samples, inner products are often pre-computed and stored in a Kernel Matrix:

Kij=k(xi,xj)=ϕ(xi)⊤ϕ(xj)\mathsf{K}_{ij} = \mathsf{k}(\mathbf{x}_i,\mathbf{x}_j) = \phi(\mathbf{x}_i)^\top \phi(\mathbf{x}_j)Kij​=k(xi​,xj​)=ϕ(xi​)⊤ϕ(xj​)
(11)

Obviously, K\mathsf KK is symmetric. If we store the matrix K\mathsf{K}K, we only need to do simple matrix look-ups and low-dimensional computations throughout the gradient descent algorithm. To make the formula more readable, we sill use the “kernel function” representation instead of the “matrix” representation. The final classifier becomes:

h(xt)=∑i=1nαik(xi,xt)h(\mathbf{x}_t)=\sum_{i=1}^n\alpha_i \mathsf{k}(\mathbf{x}_i,\mathbf{x}_t)h(xt​)=i=1∑n​αi​k(xi​,xt​)
(12)

So we can rewrite function γ\gammaγ as

γi=2(h(xi)−yi)=2[(∑j=1nαjk(xi,xj))−yi]\gamma_i = 2(h(\mathbf x_i)-y_i) = 2 \left[\left(\sum_{j=1}^n \alpha_j \mathsf{k}(\mathbf{x}_i,\mathbf{x}_j) \right)-y_i \right]γi​=2(h(xi​)−yi​)=2[(j=1∑n​αj​k(xi​,xj​))−yi​]
(13)

The gradient update becomes:

αit=αit−1−sγit−1=αit−1−2s[(∑j=1nαjk(xi,xj))−yi]\alpha_i^t = \alpha_i^{t-1} - s\gamma_i^{t-1} = \alpha_i^{t-1} - 2s\left[\left(\sum_{j=1}^n \alpha_j \mathsf{k}(\mathbf{x}_i,\mathbf{x}_j) \right)-y_i \right]αit​=αit−1​−sγit−1​=αit−1​−2s[(j=1∑n​αj​k(xi​,xj​))−yi​]

General Kernels¶

  • Linear: K(x,z)=x⊤z,ϕ(x)=x\mathsf{K}(\mathbf{x},\mathbf{z})=\mathbf{x}^\top \mathbf{z}, \phi(\mathbf{x}) = \mathbf{x}K(x,z)=x⊤z,ϕ(x)=x: The linear kernel is equivalent to just using a linear classifier, but it actually runs faster because it is a kernel and updates in a kernel way (covered in the next lecture what means to “update in a kernel way”)
  • Polynomial: K(x,z)=(1+x⊤z)d\mathsf{K}(\mathbf{x},\mathbf{z})=(1+\mathbf{x}^\top \mathbf{z})^dK(x,z)=(1+x⊤z)d, ϕ(x)\phi(\mathbf x)ϕ(x) is something similar to the ϕ\phiϕ above, but in even higher dimension. This kernel contains all polynomials of up to ddd, including something like x2z3\mathbf x^2 \mathbf z ^3x2z3, which is not covered by the kernel function in the previous section (See Equation 8).
  • Radial Basis Function (RBF) (aka Gaussian Kernel): K(x,z)=e−∥x−z∥2σ2\mathsf{K}(\mathbf{x},\mathbf{z})= e^\frac{-\|\mathbf{x}-\mathbf{z}\|^2}{\sigma^2}K(x,z)=eσ2−∥x−z∥2​: Though the polynomial kernel is in very high dimensional, it is still finite, but the dimension of RBF’s corresponding feature vector ϕ(x)\phi(\mathbf x)ϕ(x) is infinite and cannot be computed. However, an effective low dimensional approximation exists.
  • Exponential Kernel: K(x,z)=e−∥x−z∥2σ2\mathsf{K}(\mathbf{x},\mathbf{z})= e^\frac{-\| \mathbf{x}-\mathbf{z}\|}{2\sigma^2}K(x,z)=e2σ2−∥x−z∥​
  • Laplacian Kernel: K(x,z)=e−∣x−z∣σ\mathsf{K}(\mathbf{x},\mathbf{z})= e^\frac{-| \mathbf{x}-\mathbf{z}|}{\sigma}K(x,z)=eσ−∣x−z∣​
  • Sigmoid Kernel: K(x,z)=tanh⁡(ax⊤+c)\mathsf{K}(\mathbf{x},\mathbf{z})=\tanh(\mathbf{a}\mathbf{x}^\top + c)K(x,z)=tanh(ax⊤+c)

Kernel functions¶

Can any function K(⋅,⋅)\mathsf{K}(\cdot,\cdot)K(⋅,⋅) be used as a kernel?

No, the matrix K(xi,xj)\mathsf{K}(\mathbf{x}_i,\mathbf{x}_j)K(xi​,xj​) has to correspond to real inner-products of ϕ(x)\phi({\mathbf{x}})ϕ(x) - x\mathbf xx after some transformation. This is the case if and only if K\mathsf{K}K is positive semi-definite.

Prove: recall the definition of positive semi-definite: A matrix A∈Rn×nA\in \mathbb{R}^{n\times n}A∈Rn×n is positive semi-definite iff ∀q∈Rn\forall \mathbf{q}\in\mathbb{R}^n∀q∈Rn, q⊤Aq≥0\mathbf{q}^\top A\mathbf{q}\geq 0q⊤Aq≥0.

  • All Kernel functions are positive semi-definite:

    Remember Kij=ϕ(xi)⊤ϕ(xj)\mathsf{K}_{ij}=\phi(\mathbf{x}_i)^\top \phi(\mathbf{x}_j)Kij​=ϕ(xi​)⊤ϕ(xj​). So K=Φ⊤Φ\mathsf{K}=\Phi^\top\PhiK=Φ⊤Φ, where Φ=[ϕ(x1),…,ϕ(xn)]\Phi=[\phi(\mathbf{x}_1),\dots,\phi(\mathbf{x}_n)]Φ=[ϕ(x1​),…,ϕ(xn​)]. It follows that K\mathsf{K}K is p.s.d., because q⊤Kq=(Φ⊤q)2≥0\mathbf{q}^\top\mathsf{K}\mathbf{q}=(\Phi^\top \mathbf{q})^2\geq 0q⊤Kq=(Φ⊤q)2≥0.

  • All positive semi-definite matrix produces a kernel:

    if any matrix A\mathbf{A}A is p.s.d., it can be decomposed as A=Φ⊤ΦA=\Phi^\top\PhiA=Φ⊤Φ for some realization of Φ\PhiΦ.
