Comparing to GD¶

Compare the full version of minibatch SGD running time with GD, we see that the nnn is basically replaced with the max⁡(σ2μϵ,B)\max(\frac {\sigma^2} {\mu \epsilon}, B)max(μϵσ2​,B)

So there are some situations more suitable to minibatch SGD we can immediately see:

  • huge nnn: GD runs too slowly to calculate gradient on the whole dataset
  • not too big precision ϵ\epsilonϵ needed

In what situation does GD runs better? Maybe when we have too large σ2\sigma^2σ2, so the average batch gradient is not representative of the whole dataset? Is this true though? Think when we have a dataset of a very large variance (maybe all the data points are uniformly distributed on some space), so randomly sample a data point iii, ∇fi\nabla f_i∇fi​ will have almost nothing to do with E[∇fi]\mathbf E [\nabla f_i]E[∇fi​]. If this is the case, even GD cannot learn much and this learning / optimization problem just doesn’t hold itself.

Another Advantage of SGD - Hardware¶

Minibatch SGD runs significantly faster than GD not only because of the decrease in computation cost, but also because when we have too large an nnn, we will blow up the memory and have to do a lot of memory swap and other overheads.

Minibatch SGD is faster than vanilla SGD because minibatch can utilize parallelism of the hardware.

Minibatch SGD in Practice¶

When analyzing convergence and running time, we drew the batch with replacement. However, in practice, drawing the batch without replacement gives us better result.

The best practice is random reshuffle, where we randomly shuffle the whole dataset and divide it into several minibatches. So there is no duplicate elements across all the batches. This method will make our previous convergence proof absolutely not hold, because everything the previous proof built on was we assumed each batch is drawn independently, so all those expectations only depend on wtw_twt​. However, now the expectation will depend on a whole lot of stuff - what was the previous batch, the order they were drawn, ...

The term related to random shuffling is epoch, which means 1 pass through the whole dataset.

We have other methods for doing minibatch SGD:

  1. without replacement within batch level, so there can be duplicates across different batches
  2. shuffle-once: only shuffle the dataset once before start training, so each epoch has the same order
