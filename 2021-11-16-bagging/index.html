Cornell University

Bagging is an ensemble method.

Bagging Reduces Variance¶

Remember the Bias-Variance decomposition:

E[(hD(x)−y)2]⏟Error=E[(hD(x)−hˉ(x))2]⏟Variance+E[(hˉ(x)−yˉ(x))2]⏟Bias+E[(yˉ(x)−y(x))2]⏟Noise\underbrace{\mathbb{E}[(h_D(x) - y)^2]}_\mathrm{Error} = \underbrace{\mathbb{E}[(h_D(x)-\bar{h}(x))^2]}_\mathrm{Variance} + \underbrace{\mathbb{E}[(\bar{h}(x)-\bar{y}(x))^2]}_\mathrm{Bias} + \underbrace{\mathbb{E}[(\bar{y}(x)-y(x))^2]}_\mathrm{Noise}ErrorE[(hD​(x)−y)2]​​=VarianceE[(hD​(x)−hˉ(x))2]​​+BiasE[(hˉ(x)−yˉ​(x))2]​​+NoiseE[(yˉ​(x)−y(x))2]​​

Our goal is to reduce the variance term: E[(hD(x)−hˉ(x))2]\mathbb{E}[(h_D(x)-\bar{h}(x))^2]E[(hD​(x)−hˉ(x))2]. For this, we want hD→hˉh_D \to \bar{h}hD​→hˉ.

Weak law of large numbers¶

The weak law of large numbers says: for i.i.d. random variables xix_ixi​ with mean xˉ\bar{x}xˉ, we have

lim⁡m→∞1m∑i=1mxi=xˉ\lim_{m \to \infty} \frac{1}{m}\sum_{i = 1}^{m}x_i = \bar{x}m→∞lim​m1​i=1∑m​xi​=xˉ
(1)

Apply this to classifiers: Assume we have m training sets D1,D2,…,DnD_1, D_2, …, D_nD1​,D2​,…,Dn​ drawn from PnP^nPn. Train a classifier on each one and average result:

lim⁡m→∞h^=1m∑i=1mhDi=hˉ\lim_{m \to \infty} \hat{h} = \frac{1}{m}\sum_{i = 1}^m h_{D_i} = \bar{h}m→∞lim​h^=m1​i=1∑m​hDi​​=hˉ
(2)

We refer to this average of multiple classifiers as an ensemble of classifiers. Good news: If h^→hˉ\hat{h}\rightarrow \bar{h}h^→hˉ the variance component of the error must also vanish, i.e. E[(h^(x)−hˉ(x))2]→0\mathbb{E}[(\hat{h}(x)-\bar{h}(x))^2]\rightarrow 0E[(h^(x)−hˉ(x))2]→0. However, the problem is that we don’t have mmm data sets D1,….,DmD_1, …., D_m D1​,….,Dm​. We only have a single DDD.

Solution: Bagging (Bootstrap Aggregating)¶

We need to sample a total mmm dataset DiD_iDi​, each of size n=∣D∣n = |D|n=∣D∣.

To do this, simulate drawing from PPP by drawing uniformly with replacement from the set DDD.

Mathematically, let Q(X,Y∣D)Q(X,Y|D)Q(X,Y∣D) be a probability distribution that picks a training sample (xi,yi)(\mathbf{x}_i,y_i)(xi​,yi​) from DDD uniformly at random. i.e. ∀(xi,yi)∈D,  Q((xi,yi)∣D)=1n\forall (\mathbf{x_i}, y_i)\in D, \; Q((\mathbf{x_i}, y_i)|D) = \frac{1}{n}∀(xi​,yi​)∈D,Q((xi​,yi​)∣D)=n1​ with n=∣D∣n=|D|n=∣D∣. Notice we cannot use the W.L.L.N here: h^D=1m∑i=1mhDi↛hˉ\hat{h}_D = \frac{1}{m}\sum_{i = 1}^{m}h_{D_i}\nrightarrow \bar{h}h^D​=m1​∑i=1m​hDi​​↛hˉ because these data points are not drawn i.i.d. from the original distribution PPP, where hDh_DhD​ comes from. Even though they are drawn i.i.d from the distribution QQQ (which is conditioned on DDD.) However, in practice bagging still reduces variance very effectively.

Analysis¶

Although we cannot prove that the new samples are i.i.d., we can show that they are drawn from the original distribution PPP, namely Q(X=xi)=P(X=xi)Q(X=x_i)=P(X=x_i)Q(X=xi​)=P(X=xi​). A proof can be found at the original lecture notes.

Bagging summarized¶

  1. Sample mmm data sets D1,…,DmD_1,\dots,D_mD1​,…,Dm​ from DDD with replacement.
  2. For each DjD_jDj​ train a classifier hj()h_j()hj​()
  3. The final classifier is h(x)=1m∑j=1mhj(x)h(\mathbf{x})=\frac{1}{m}\sum_{j=1}^m h_j(\mathbf{x})h(x)=m1​∑j=1m​hj​(x).

In practice larger mmm results in a better ensemble, but at some point you will obtain diminishing returns. Note that setting mmm unnecessarily high will only slow down your classifier but will not increase the error of your classifier.

Advantages of Bagging¶

  • Easy to implement

  • Reduces variance, so has a strong beneficial effect on high variance classifiers.

  • We can obtain a mean score and variance as the prediction is an average of many classifiers. Variance can be interpreted as the uncertainty of the prediction. Especially in regression tasks, such uncertainties are otherwise hard to obtain. For example, imagine the prediction of a house price is $300,000. If a buyer wants to decide how much to offer, it would be very valuable to know if this prediction has standard deviation +-$10,000 or +-$50,000.

  • Bagging provides an unbiased estimate of the test error- the out-of-bag error. The idea is that for each training point, there are very likely to be some data sets DkD_kDk​ that did not pick this point. If we average the classifiers hkh_khk​ of all such data sets, we obtain a classifier that was not trained on (xi,yi)(\mathbf{x}_i,y_i)(xi​,yi​) ever. To these classifiers, this point is equivalent to a test sample. If we compute the error of all these points, we obtain an estimate of the true test error.

    More formally, for each training point (xi,yi)∈D(\mathbf{x}_i,y_i)\in D(xi​,yi​)∈D let Si={k∣(xi,yi)∉Dk}S_i=\{k| (\mathbf{x}_i,y_i)\notin D_k\}Si​={k∣(xi​,yi​)∈/Dk​} - in other words SiS_iSi​ - the set of all the training sets DkD_kDk​ that do not contain (xi,yi)(\mathbf{x}_i,y_i)(xi​,yi​).     This is an estimate of the test error, because for each training point we used the subset of classifiers that never saw that training point during training. if mmm is sufficiently large, the fact that we take out some classifiers has no significant effect and the estimate is pretty reliable.

Random Forest¶

A Random Forest is essentially bagged decision trees, with a slightly modified splitting criteria - don’t use all features.

The algorithm works as follows:

  1. Sample mmm data sets D1,…,DmD_1,\dots,D_mD1​,…,Dm​ from DDD with replacement.
  2. For each DjD_jDj​ train a full decision tree hjh_jhj​ (max-depth=∞\infty∞) with one small modification: before each split randomly subsample k≤dk\leq dk≤d features (without replacement) and only consider these features for your split. This further increases the variance of the trees. A good choice for kkk is k=dk=\sqrt{d}k=d​
  3. The final classifier is h(x)=1m∑j=1mhj(x)h(\mathbf{x})=\frac{1}{m}\sum_{j=1}^m h_j(\mathbf{x})h(x)=m1​∑j=1m​hj​(x).

The randomness in random forest comes from the random sampling in step 1 and the random feature selection in step 2.

The Random Forest is one of the best and easiest to use model. There are two reasons:

  • It only has two hyper-parameters, mmm and kkk and is extremely insensitive to both of these. You can set mmm as large as you can afford and a popular choice of kkk is d\sqrt dd​
  • Decision trees do not require a lot of preprocessing, so the features can be of different scale, magnitude, or slope. This is highly advantageous with heterogeneous data, for example with features like blood pressure, age, gender, ... each of which is recorded in completely different units.

Useful variants of Random Forests:

  • Split each training set into two partitions Dl=DlA∪DlBD_l=D_l^A\cup D_l^BDl​=DlA​∪DlB​. Build the tree on DlAD_l^ADlA​ and estimate the leaf labels on DlBD_l^BDlB​. You must stop splitting if a leaf has only a single point in DlBD_l^BDlB​ in it. This has the advantage that each tree and also the RF classifier become consistent.
  • Do not grow each tree to its full depth, instead prune based on the leave out samples. This can further improve your bias/variance trade-off. (An easy way to prune decision trees is to start from the bottom and then to remove splits and check to see if it affects our error too much. If it doesn’t we remove that split.)
