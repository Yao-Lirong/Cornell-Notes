Back Propagation

Cornell University
August 31, 2022

Reverse Mode Automatic Differentiation¶

We need to fix an output in the NN. For most cases, we just use the final output ℓ\ellℓ of the NN (ℓ\ellℓ stands for loss). For each intermediate output value yyy, we want to compute partial derivatives ∂ℓ∂y\frac{\partial \ell}{\partial y}∂y∂ℓ​.

Same as before: we replace each number/array yyy with a pair, except this time the pair is not (y,∂y∂x)(y, \frac{\partial y}{\partial x})(y,∂x∂y​) but instead (y,∇yℓ)(y, \nabla_y \ell)(y,∇y​ℓ) where ℓ\ellℓ is our final output (loss of NN). Observe that ∇yℓ\nabla_y \ell∇y​ℓ always has the same shape as yyy.

Deriving Backprop from the Chain Rule¶

Single Variable¶

Suppose that the output ℓ\ellℓ can be written as

ℓ=f(u),  u=g(y)\ell = f(u), \; u = g(y)ℓ=f(u),u=g(y)
(1)

Here, ggg represents the immediate next operation that takes yyy (producing a new intermediate uuu), and fff represents all the rest of the computation between uuu and ℓ\ellℓ. By the chain rule,

∂ℓ∂y=∂ℓ∂u∂u∂y=g′(y)⋅∂ℓ∂u\begin{align} \frac{\partial \ell}{\partial y} &= \frac{\partial \ell}{\partial u} \frac{\partial u}{\partial y} \\ &= g'(y) \cdot \frac{\partial \ell}{\partial u} \end{align}∂y∂ℓ​​=∂u∂ℓ​∂y∂u​=g′(y)⋅∂u∂ℓ​​​
(2)

Therefore, to compute ∂ℓ∂y\frac{\partial \ell}{\partial y}∂y∂ℓ​, in addition to ∂ℓ∂u\frac{\partial \ell}{\partial u}∂u∂ℓ​ - the derivative of all the other intermediate computation, we only need to use “local” information g′(y)g'(y)g′(y) that’s available in the program around where yyy is computed and used. That is, if ignore ∂ℓ∂u\frac{\partial \ell}{\partial u}∂u∂ℓ​, we can compute ∂ℓ∂y\frac{\partial \ell}{\partial y}∂y∂ℓ​ together when we compute yyy.

Multi Variable¶

More generally, suppose that the output ℓ\ellℓ can be written as

ℓ=F(u1,u2,…,uk),  ui=gi(y)\ell = F(u_1, u_2, \ldots, u_k), \; u_i = g_i(y)ℓ=F(u1​,u2​,…,uk​),ui​=gi​(y)
(3)

Here, the g1,g2,…,gkg_1, g_2, \ldots, g_kg1​,g2​,…,gk​ represent the immediate next operations that take yyy; the uiu_iui​ are the output by these operations, and FFF represents all the rest of the computation between uuu and ℓ\ellℓ. By the multivariate chain rule,

∂ℓ∂y=∑i=1k∂ℓ∂ui∂ui∂y∇yℓ=∑i=1kDgi(y)T∇uiℓ\begin{align} \frac{\partial \ell}{\partial y} &= \sum_{i=1}^k \frac{\partial \ell}{\partial u_i} \frac{\partial u_i}{\partial y}\\ \nabla_y \ell &= \sum_{i=1}^k D g_i(y)^T \nabla_{u_i} \ell \end{align}∂y∂ℓ​∇y​ℓ​=i=1∑k​∂ui​∂ℓ​∂y∂ui​​=i=1∑k​Dgi​(y)T∇ui​​ℓ​​
(4)

Note here we are talking about the more general case, where ℓ,u,y\ell, u, yℓ,u,y are all vectors. So gig_igi​ takes in a vector yyy and outputs a vector uiu_iui​ When we take the derivative of gig_igi​ at yyy, we get a matrix and we call this matrix Dgi(y)D g_i(y)Dgi​(y)

Again, we see that we can compute ∇yℓ\nabla_y \ell∇y​ℓ using only ∇uiℓ\nabla_{u_i} \ell∇ui​​ℓ and “local” information Dgi(y)D g_i(y)Dgi​(y).

Computing Backprop¶

When to Compute?¶

Note the ℓ\ellℓ value is computed as follows: y→u→ℓy \to u \to \elly→u→ℓ So when we are computing yyy , which is also when we ideally want to compute ∂ℓ∂y\frac{\partial \ell}{\partial y}∂y∂ℓ​, the value of uuu is not yet available. Without uuu, we cannot compute ∂ℓ∂u\frac{\partial \ell}{\partial u}∂u∂ℓ​, which is a necessary part to get ∂ℓ∂y\frac{\partial \ell}{\partial y}∂y∂ℓ​. Therefore we cannot compute ∂ℓ∂y\frac{\partial \ell}{\partial y}∂y∂ℓ​ along with yyy

Therefore, we cannot compute ∂ℓ∂y\frac{\partial \ell}{\partial y}∂y∂ℓ​ until after we’ve computed everything between yyy and ℓ\ellℓ. So the solution is to remember the order in which we computed stuff in between yyy and ℓ\ellℓ, and then compute their gradients in the reverse order.

Computational Graph¶

We draw a graph to indicate the computational dependencies, just like the y→u→ℓy \to u \to \elly→u→ℓ above:

  • node represents an scalar/vector/tensor/array
  • Edges represent dependencies, where y→uy \rightarrow uy→u means that yyy was used to compute uuu

Algorithm¶

Generally speaking,

  1. For each tensor, initialize a gradient “accumulator” to 0. Set the gradient of ℓ\ellℓ with respect to itself to 1.

  2. For each tensor y′y'y′ that ℓ\ellℓ depends on, in the reverse order of computation, compute ∇y′ℓ\nabla_{y'} \ell∇y′​ℓ - derivative of ℓ\ellℓ with respect to y′y'y′. This is obtainable because when we get to y′y'y′, all the intermediate value uuu between y′y'y′ and ℓ\ellℓ are already computed.

    Once this step is called for yyy, its gradient will be fully manifested in its gradient accumulator.

In a computational graph, this means

  1. For each tensor, initialize a gradient “accumulator” to 0. Set the gradient of ℓ\ellℓ with respect to itself to 1.

  2. For each tensor y′y'y′ pointing to ℓ\ellℓ, start from ℓ\ellℓ and go back along the path, compute ∇y′ℓ\nabla_{y'} \ell∇y′​ℓ - derivative of ℓ\ellℓ with respect to y′y'y′. Specifically, we compute this value by looking at all its direct descendants uiu_iui​ where ∇y′ℓ=∑i=1k∂ℓ∂ui∂ui∂y′\nabla_{y'} \ell = \sum_{i=1}^k \frac{\partial \ell}{\partial u_i} \frac{\partial u_i}{\partial y'}∇y′​ℓ=∑i=1k​∂ui​∂ℓ​∂y′∂ui​​ We can do this because uuu are the nodes between y′y'y′ and ℓ\ellℓ, when we get to y′y'y′, all uuu values are already computed.

    Once this step is called for yyy, computation is done.
